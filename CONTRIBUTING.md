# Contributing to JaanchLo

Thank you for helping protect people from scams. A few ground rules keep this tool trustworthy.

## The one hard rule
Every change must keep the CI gate green:
```bash
node engine/gate.mjs
```
This re-extracts the engine from the app, runs the engine test suites, checks dataset integrity, and enforces the evaluation thresholds (scam recall, danger recall, and a strict benign false-danger ceiling). PRs that regress any threshold will not merge.

## The app is the source of truth
The detector and all verdict copy live in `app/jaanchlo_app.html`. Do not hand-edit `dataset/engine.mjs` or `engine/explain_data.mjs`; they are regenerated by `node engine/extract.mjs`. Change the app, then re-extract.

## Adding scam patterns or languages
- Add signals/knockouts in the app engine, add reason strings in **both** English and Hindi (parity is tested).
- Add labeled templates in `dataset/augment.mjs` with an authored `expected` tier.
- Regenerate and re-run the gate. New benign controls are especially welcome: protecting the false-danger rate is the highest priority.

## Principles
- Never claim certainty. Verdicts are a second opinion; the UI must keep the 1930 helpline visible.
- Elder-first: plain language, large targets, no jargon.
- Privacy by default: no logging or storage of user submissions.
- No em dashes in user-facing copy (project style).
